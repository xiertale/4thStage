# Документ учебной программы курса "Веб-разработка"
Цель курса: Формирование у студентов полного цикла компетенций, необходимых для создания современных, производительных и доступных веб-приложений, и подготовка основы для реализации индивидуального выпускного проекта.

## 1. Результат обучения: Индивидуальный проект

В рамках курса каждый студент поэтапно создает и наполняет функционалом основу для своего индивидуального проекта, который включает в себя:

- Backend API на Node.js с использованием TypeORM.
- Базу данных с продуманной схемой и связями.
- Frontend-приложение на React/Next.js для взаимодействия с пользователем.
- Адаптивный и производительный интерфейс, соответствующий современным стандартам.
- Систему контроля версий для своего проекта (Git-репозиторий).

## 2. Осваиваемые знания и навыки


### Верстка (HTML & CSS)
- HTML5: Синтаксис, базовые теги, формы, валидация.
- Семантическая верстка: Понимание и правильное применение семантических тегов (header, nav, main, section, article, aside, footer).
- CSS: Селекторы, каскадность, наследование, позиционирование, блочная модель.
- CSS Flexbox: Понимание модели и свойств для создания одномерных макетов.
- CSS Grid: Понимание модели и свойств для создания сложных двумерных макетов.
- CSS переменные (Custom Properties): Создание и использование переменных для цветов, размеров и других значений.

### Система контроля версий (Git)
- Понимание основных принципов и приемов работы с Git.
- Практические навыки работы с собственным репозиторием (создание, коммиты, ветвление, пушинг).

### Backend-разработка
- Теория: Глубокое понимание концепции API (Application Programming Interface).
- Практика: Навык создания API-маршрутов (routes) для операций CRUD: получения, добавления и удаления данных.

### Работа с базами данных (БД)
- ORM: Практическая работа с TypeORM.
- Создание сущностей (Entities).
- Настройка и понимание связей между сущностями: один-ко-многим (1:n), один-к-одному (1:1), многие-ко-многим (n:n).

### Инструменты:
- Навык вызова API для тестирования его работоспособности (например, через Rest Plugin, Postman). [HTTP запросы](docs/http.md)
- Умение пользоваться инструментами для прямого взаимодействия с БД. [БД SQLite](docs/db.md)

### Взаимодействие с API (HTTP & Fetch)
- HTTP: Понимание структуры протокола: методы запросов (GET, POST, PUT, DELETE), статусы ответов, заголовки (headers), структура запроса и ответа.
- Fetch: Обязательное знание принципов работы и применение стандартной функции fetch() для выполнения HTTP-запросов на клиенте.

### Фронтенд-разработка (React)
- Компоненты: Понимание концепции компонентов и умение их создавать и использовать.
- Хуки (Hooks): Понимание концепции хуков, умение использовать встроенные хуки (useState, useEffect) и создавать собственные.
- Формы: Понимание работы с формами, валидация.

### Фреймворк Next.js
- Деплой: Умение развернуть проект из собственного или общего репозитория.
- Страницы и маршрутизация: Понимание и использование файловой маршрутизации (File-based Routing), умение создавать страницы.
- Layout: Понимание концепции шаблона (Layout), его структуры и умение добавлять новые пункты в навигационное меню.

### Инструменты разработки и отладка
- Владение основными методами отладки кода: использование debugger и console.log.
- Навык мониторинга и анализа HTTP-запросов через инструменты разработчика в браузере (DevTools, вкладка "Network").
- Использование браузерных DevTools для тестирования адаптивности и анализа производительности рендеринга.

[Debug - отладка, средства разработки браузера](docs/debug.md)

### Управление состоянием данных (TanStack Query)
- Хуки: Понимание принципов работы хуков в TanStack Query для получения и изменения данных.
- Запросы (Queries): Получение и кэширование данных с сервера.
- Мутации (Mutations): Выполнение операций по изменению данных (создание, обновление, удаление).
- Инструменты: Использование Devtools TanStack Query для отладки состояния кэша.
  
[TanStackQuery - работа с данными в приложении](docs/tanstack-query.md)


### Полный цикл работы с данными (Ключевая компетенция)
Студент должен понимать и уметь объяснить полную цепочку данных в приложении:
- От БД к UI: Что происходит от момента выполнения запроса к базе данных на бэкенде до момента отображения этих данных на странице в браузере.
- От UI к БД: Что происходит от момента отправки пользователем данных через форму на фронтенде до момента появления новой записи в базе данных.
- Объяснение на коде: Умение продемонстрировать и объяснить эту цепочку, ссылаясь на конкретные участки кода в своем проекте (сущность БД, API-роут, запрос с фронтенда, компонент).


### Производительность и пользовательский опыт (Core Web Vitals)
- Понимание метрик:
  - LCP (Largest Contentful Paint): Понимание важности скорости загрузки основного контента.
  - FID (First Input Delay) / INP (Interaction to Next Paint): Понимание важности отзывчивости интерфейса.
  - CLS (Cumulative Layout Shift): Понимание важности визуальной стабильности при загрузке страницы.

- Измерение и анализ: Умение пользоваться инструментами для измерения этих метрик (Lighthouse, Google PageSpeed Insights, Web Vitals Extension).

- Базовые принципы оптимизации: Знание основных подходов для улучшения показателей (оптимизация изображений, эффективное подключение ресурсов, минимизация работы в основном потоке).

### Адаптивная верстка
- Теория: Понимание принципов и важности создания адаптивных интерфейсов (Responsive Web Design).
- Практика: Владение основными технологиями для реализации адаптивности:
  - CSS Media Queries.
  - Относительные единицы измерения (%, vw, vh, rem, em).
  - Flexbox и CSS Grid для создания адаптивных макетов.
- Mobile-First подход: Понимание концепции и её преимуществ.


### Фундаментальные основы JavaScript
- Прототипное наследование: Понимание прототипной модели JavaScript, умение работать с цепочкой прототипов. Знание отличий от классового наследования.
- Event Loop: Понимание модели выполнения JavaScript (стек вызовов, очередь задач, микротаски и макротаски). Объяснение асинхронности и неблокирущих операций.
- Hoisting (Всплытие): Понимание механизма всплытия объявлений переменных (var, let, const) и функций. Знание различий между разными типами объявлений.
- Critical Rendering Path (Критический путь рендеринга): Понимание последовательности шагов, которые браузер выполняет для преобразования HTML, CSS и JavaScript в пиксели на экране. Знание как оптимизировать CRP для ускорения первоначальной отрисовки страницы.

# Критерии оценки по 5-балльной шкале

## Оценка «5» (Отлично)
### Проект:
- Индивидуальный проект полностью завершен, развернут и стабильно работает.
- Код чистый, хорошо структурирован и соответствует лучшим практикам.
- Репозиторий ведется идеально: понятные коммиты, актуальный README с описанием и инструкцией по запуску.
- Верстка: Семантически правильная, кроссбраузерная, полностью адаптивная. Используются Flexbox/Grid там, где это уместно. CSS переменные применяются для поддержания единого стиля.

### Знания и навыки:

- Backend & БД: Создано полнофункциональное API (CRUD). Схема БД продумана, все связи реализованы корректно. TypeORM используется эффективно.
- Frontend (React/Next.js): Приложение отзывчивое, интуитивно понятное и полностью адаптивное. Компоненты переиспользуемы, хуки применяются уместно. Роутинг в Next.js реализован без ошибок.
- Производительность: Понимание Core Web Vitals продемонстрировано на практике: показатели LCP, CLS и INP у проекта находятся в "зеленой" зоне. Критический путь рендеринга оптимизирован.
- Работа с данными (TanStack Query): Данные кэшируются корректно, мутации обрабатывают состояния загрузки и ошибок. Инструменты разработки используются для отладки.
- Фундаменты JS: Студент может уверенно объяснить, как в его проекте работают Event Loop, прототипное наследование, и как его код влияет на Critical Rendering Path.
- Полный цикл: Студент детально объясняет всю цепочку данных от фронтенда до БД и обратно, ссылаясь на конкретный код своего проекта.
